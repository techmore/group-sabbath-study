# Data Architecture for Original Languages

> **Sources & Attribution**
>
> - **Hebrew (Old Testament)**: Text is primarily drawn from the [Sefaria](https://www.sefaria.org) API, using their high-quality digital editions of the Masoretic Text (e.g. "Miqra according to the Masorah").
> - **Greek (New Testament)**: Derived from the diplomatic transcription of **Codex Sinaiticus** (4th century), one of the most important surviving manuscripts of the Bible.
>   - Official project repository: https://github.com/itsee-birmingham/codex-sinaiticus
>   - The raw transcription file used: `FINAL_TRANSCRIPTION_version104.xml`
>   - License note: The Codex Sinaiticus Project makes the transcription available for non-commercial re-use with attribution (Creative Commons BY-NC-SA style).
>
> All curated original-language content in this project must include clear attribution to these sources.

## Goals
- Support clean, full-verse Hebrew, Aramaic, and Koine Greek in the "Full Original Text" toggle.
- Keep the main application (`index.html` + core config) lightweight and easy for groups to fork.
- Make adding and maintaining original language content sustainable over 52 weeks.
- Use Codex Sinaiticus transcription (provided as FINAL_TRANSCRIPTION_version104.xml) as the authoritative source for Greek.

## Current State (as of this plan)

### Hebrew (OT)
- Source: Sefaria API (high-quality, normalized Masoretic text)
- Storage: `data/original-languages.json` (curated per week)
- App integration: Working via `findOriginalLanguageExamplesForRef()`

### Greek (NT)
- Source: Codex Sinaiticus TEI XML (FINAL_TRANSCRIPTION_version104.xml)
  - 4th century majuscule manuscript
  - Diplomatic transcription (very detailed, includes layout + scribal corrections)
  - ~29MB, 142k lines, complex TEI structure
- Status: Extracted and normalized into `data/greek/nt/*.json` for all NT books.
- The raw XML is still the authoritative source file, but the usable extracted layer is now complete.

## Recommended Architecture

```
project/
├── index.html                 # UI + STUDY_CONFIG (plan structure, not raw text)
├── data/
│   ├── original-languages.json   # Curated verses actually shown per week
│   │                               (both Hebrew + Greek, keyed by week)
│   ├── greek/
│   │   └── nt/
│   │       ├── matthew.json      # Clean extracted text per book
│   │       ├── john.json
│   │       └── ...
│   └── hebrew/                   # (future) if we want per-book Hebrew too
├── scripts/
│   └── extract-greek-nt.js       # One-time / occasional processing script
└── docs/
    └── DATA-ARCHITECTURE.md
```

### Two-Layer Model (Important)

1. **Raw / Extracted Layer** (`data/greek/nt/*.json`)
   - Full normalized text per book/chapter/verse from Sinaiticus.
   - Generated by script from the big XML.
   - Rarely edited by hand.

2. **Curated Layer** (`data/original-languages.json`)
   - Only the specific verses the study plan actually wants to highlight each week.
   - This is what the app primarily loads and displays.
   - Human-maintained (with help from the extracted layer).

Generated files in `data/greek/nt/*.json` automatically include source metadata pointing to the Codex Sinaiticus GitHub repository.

This prevents the app from becoming bloated with every verse in the New Testament.

## Verse Data Shape (proposed)

For Hebrew, Aramaic, and Greek in the curated file:

```json
{
  "22": [
    {
      "ref": "John 1:1",
      "language": "Greek",
      "text": "Ἐν ἀρχῇ ἦν ὁ λόγος...",
      "translit": "En archē ēn ho logos...",
      "meaning": "In the beginning was the Word..."
    }
  ]
}
```

For the extracted per-book files, we can use a simpler nested structure:

```json
{
  "1": {
    "1": {
      "text": "...",
      "translit": "..."
    }
  }
}
```

## Processing Pipeline (Greek)

1. Run `node scripts/extract-greek-nt.js` (once or when source updates).
2. Script parses the large TEI XML (needs to handle size — streaming recommended).
3. Outputs one clean JSON file per NT book into `data/greek/nt/`.
4. Later, when curating a week, copy/paste or reference the desired verses into `original-languages.json`.

Current status:
- The one-time extraction step has already been completed for the full NT set in this repo.
- Re-run the script only if the transcription source changes or the extraction logic improves.

## App Integration

- `findOriginalLanguageExamplesForRef()` resolves curated reading refs from `data/original-languages.json`.
- `renderScriptureBlock()` shows Greek whenever a curated NT week includes it.
- The UI already labels Greek as `Koine Greek • Codex Sinaiticus`.
- The checkbox label / help text already mentions Hebrew, Aramaic, and Greek when present.

## Phased Implementation

**Phase 1**: Set up structure + proof-of-concept extraction for 1-2 books (e.g. John + Matthew).

**Phase 2**: Full extraction script that can process the entire New Testament reliably.

**Phase 3**: Curate Greek verses into `original-languages.json` for weeks that will have NT content.

**Phase 4**: UI polish (proper Greek font stack, better transliteration quality, notes about the manuscript source).

## Open Questions / Decisions

- Do we want to eventually support multiple Greek witnesses (e.g. also a Nestle-Aland based text), or stick with Sinaiticus as our "original" for this study?
- How much transliteration do we want to provide? (Sinaiticus has no accents/breathing in the same way modern editions do.)
- Should we also extract the Septuagint portions from this same XML for consistency on OT readings?
